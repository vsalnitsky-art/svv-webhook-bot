# SVV Webhook Trading Bot 🚀

Автоматизированный торговый бот для Bybit с поддержкой вебхуков, мониторингом позиций и аналитикой.

## 📋 Возможности

- ✅ Автоматическое выполнение сделок через webhook
- ✅ Трейлинг стоп-лосс для защиты прибыли
- ✅ Разделенные тейк-профиты (Split TP)
- ✅ Мониторинг активных позиций в реальном времени
- ✅ Расчет RSI и рыночного давления
- ✅ Подробная аналитика P&L
- ✅ Защита API ключей с помощью шифрования
- ✅ Веб-интерфейс для мониторинга

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка API ключей

#### Вариант A: Незашифрованные ключи (для быстрого тестирования)

Установите переменные окружения:
```bash
export BYBIT_API_KEY="ваш_api_key"
export BYBIT_API_SECRET="ваш_api_secret"
```

#### Вариант B: Зашифрованные ключи (рекомендуется для продакшена)

1. Сгенерируйте ключ шифрования:
```bash
python -c "from config import generate_encryption_key; generate_encryption_key()"
```

2. Зашифруйте ваши API ключи:
```bash
python -c "from config import encrypt_credentials; encrypt_credentials('YOUR_API_KEY', 'YOUR_API_SECRET', 'YOUR_ENCRYPTION_KEY')"
```

3. Установите зашифрованные переменные окружения:
```bash
export ENCRYPTION_KEY="ваш_ключ_шифрования"
export BYBIT_API_KEY_ENCRYPTED="зашифрованный_api_key"
export BYBIT_API_SECRET_ENCRYPTED="зашифрованный_api_secret"
```

### 3. Запуск бота

```bash
python main_app.py
```

Бот будет доступен по адресу: `http://localhost:10000`

## 🌐 Endpoints

### Webhook
- **POST** `/webhook` - Получение торговых сигналов
  ```json
  {
    "action": "Buy",
    "symbol": "BTCUSDT",
    "riskPercent": 5.0,
    "leverage": 20,
    "takeProfit": 50000,
    "takeProfitPercent": 3.0
  }
  ```

### Веб-интерфейс
- **GET** `/` - Перенаправление на сканер
- **GET** `/scanner` - Мониторинг активных позиций
- **GET** `/report` - Аналитика P&L
  - Параметры: `?days=7` или `?days=30`

### Системные
- **GET** `/health` - Проверка здоровья сервера

## 📊 Функции торговли

### Управление рисками
- Автоматический расчет размера позиции на основе процента риска
- Минимальный баланс для торговли: 5 USDT
- Комиссия учитывается в расчетах (0.98 от доступного баланса)

### Трейлинг стоп-лосс
Автоматически настраивается в зависимости от актива:
- BTC, ETH, BNB: 0.5%
- SOL, XRP, ADA: 1.5%
- Остальные: 3.0%

### Тейк-профит
- 50% позиции закрывается на 40% расстояния до цели
- 50% позиции закрывается на целевой цене
- По умолчанию: +3% от точки входа

## 📈 Мониторинг

### Scanner
Отслеживает активные позиции и отображает:
- Текущую цену входа
- Нереализованный P&L
- RSI (Индекс относительной силы)
- Рыночное давление (Volume Pressure)

### Report
Предоставляет детальную аналитику:
- Общий P&L за период
- Win rate (процент успешных сделок)
- График изменения баланса
- Топ монет по прибыльности
- Детальная история сделок

## 🚢 Деплой на Render

1. Создайте новый Web Service на Render
2. Подключите ваш репозиторий
3. Установите Environment Variables:
   ```
   BYBIT_API_KEY=your_key
   BYBIT_API_SECRET=your_secret
   ```
   или используйте зашифрованные версии
4. Render автоматически обнаружит `render.yaml`

### render.yaml
```yaml
services:
  - type: web
    name: svv-webhook-bot
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn main_app:app
    envVars:
      - key: PORT
        value: 10000
```

## 🛠️ Troubleshooting

### Бот не запускается
1. Проверьте установку всех зависимостей
2. Убедитесь, что API ключи правильно установлены
3. Проверьте логи на наличие ошибок

### Сделки не выполняются
1. Проверьте баланс (минимум 5 USDT)
2. Убедитесь, что API ключи имеют права на торговлю
3. Проверьте формат webhook запроса

### Ошибки подключения к Bybit
1. Проверьте интернет-соединение
2. Убедитесь, что Bybit API доступен
3. Проверьте правильность API ключей

## 📚 Дополнительная документация

### API ключи Bybit
1. Зайдите в Bybit → API Management
2. Создайте новый API ключ
3. Установите разрешения: Trade, Position
4. Сохраните ключи безопасно

### TradingView Webhook
1. Создайте алерт в TradingView
2. В настройках алерта выберите Webhook URL
3. Укажите URL вашего сервера: `https://your-app.onrender.com/webhook`
4. В сообщении используйте JSON формат
