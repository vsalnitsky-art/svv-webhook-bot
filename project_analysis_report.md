## ❌ УСТАРЕВШИЕ ФАЙЛЫ (Старый код без БД)

### 1. `app.py` (957 строк) - УДАЛИТЬ
**Проблемы:**
- Хранит данные сканера в памяти (`pump_alerts = []`)
- Все данные теряются при перезапуске
- Дублирует весь функционал бота
- НЕ использует `scanner.py` и базу данных
- НЕ использует `statistics_service`
- Конфликтует с новой архитектурой

**Что делает:**
- Реализует свой сканер внутри файла (строки 220-419)
- Хранит историю в списках `pump_alerts`, `coin_24h_data`
- Рендерит страницу сканера `/scanner` со СТАРЫМИ данными
- Дублирует все торговые функции

**Почему нужно удалить:**
```python
# app.py использует устаревший подход:
pump_alerts = []  # Данные в памяти!
coin_24h_data = {}  # Теряются при перезапуске!

# Вместо:
stats_service.get_whale_signals()  # Из БД!
```

## 🔥 КРИТИЧЕСКАЯ ПРОБЛЕМА: Сканер работает на старом коде

### Текущая ситуация:
```
User открывает /scanner
         ↓
   Загружается app.py (СТАРЫЙ КОД)
         ↓
   Данные из памяти (pump_alerts)
         ↓
   ❌ НЕТ связи с scanner.py
   ❌ НЕТ связи с базой данных
   ❌ Данные теряются при перезапуске
```

### Должно быть:
```
User открывает /scanner
         ↓
   Загружается main_app.py (НОВЫЙ КОД)
         ↓
   scanner = EnhancedMarketScanner()
         ↓
   Данные из stats_service → База данных
         ↓
   ✅ Постоянное хранение
   ✅ Аналитика и агрегация
   ✅ API endpoints для данных
```

## 🔧 ЧТО ДЕЛАТЬ

```bash

```

### Шаг 2: Убедиться что используется main_app.py
В `render.yaml` должно быть:

Или в Render Dashboard → Settings → Build & Deploy:

```
Start Command: gunicorn main_app:app
```
