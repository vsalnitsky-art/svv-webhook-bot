# infosite — інформаційна панель бота (read-only)

Статичний сайт. Лише **показує** стан бота, читаючи його JSON API. Жодного
бекенду, жодної логіки торгівлі, жодних кнопок керування.

## Що лежить у цій теці (це і є все, що йде на хостинг)

```
infosite/
├── index.html    # сторінка
├── style.css     # стилі
├── app.js        # логіка: опитує API бота й малює таблиці
├── config.js     # ⚙️ єдине, що треба налаштувати: адреса бота
└── README.md     # ця інструкція (на хостинг не обов'язкова)
```

## Налаштування

Відредагуй **`config.js`**:

```js
window.INFOSITE_CONFIG = {
  BOT_API_BASE: "https://адреса-твого-бота",  // де відповідає API бота
  REFRESH_MS: 5000,                            // частота оновлення
};
```

- Якщо сайт віддається **тим самим сервером**, що й бот — лиши `BOT_API_BASE: ""`.
- Якщо бот на **окремому** хості — встав його повний URL.

## Деплой

Це звичайні статичні файли — підійде будь-який статичний хостинг
(Netlify, Vercel, GitHub Pages, Render Static Site, nginx). Просто поклади
вміст теки `infosite/` у корінь сайту.

Локальна перевірка:
```bash
cd infosite
python -m http.server 8080
# відкрий http://127.0.0.1:8080
```

## ⚠️ CORS (якщо бот на іншому домені)

Браузер заблокує запити з сайту на API бота, якщо домени різні й бот не
віддає заголовок `Access-Control-Allow-Origin`. Варіанти:
1. Віддавати сайт із того самого домену, що й бота (`BOT_API_BASE: ""`) — CORS не потрібен.
2. Або увімкнути CORS на боці бота для GET-маршрутів `/api/*`.

## Які дані показує

- `GET /api/fuel-filter/state` → банер ₿ BTC, ❤️ черга на вхід, 💰 funding-монети.
- `GET /api/trades?status=OPEN` → відкриті угоди.
- `GET /api/trades?status=CLOSED` → останні закриті угоди.

Тільки GET. Сайт ніколи не викликає POST-маршрути бота.
