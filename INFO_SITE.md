# INFO_SITE.md — бриф інформаційного сайту (read-only дашборд)

> Цей файл — завдання для ОКРЕМОЇ сесії Claude Code. Спершу прочитай `CLAUDE.md`,
> потім цей бриф. Сайт ТІЛЬКИ показує інформацію роботи бота — жодного керування.

## Мета

Окремий веб-сайт, що в реальному часі відображає, як працює бот: чергу на вхід,
таймери, угоди, funding-монети, статус BTC/двигуна. Лише перегляд, без кнопок,
що змінюють стан бота.

## Архітектура (рекомендована — узгодь з користувачем перед стартом)

- **Окремий read-only застосунок** (новий малий Flask або статичний фронт +
  кілька проксі-маршрутів), на ІНШОМУ порту, ніж бот. Повна ізоляція: код бота
  (`detection/`, `trade_manager`, `fuel_filter`) НЕ редагувати.
- **Джерело даних — JSON API бота** (HTTP), не пряме читання БД. Бот = єдине
  джерело правди. Базовий URL бота брати з конфіга/env (напр. `BOT_API_BASE`).
- Розташування коду: окрема тека, напр. `infosite/` (app + templates + static),
  щоб не змішувати з ботом.

## Головне джерело даних

`GET {BOT_API_BASE}/api/fuel-filter/state` — повертає `ff.get_state()`: чергу,
таймери, funding-таблицю, банер BTC, статуси. Це основа сайту. Додатково за
потреби: `/api/stats`, `/api/trades`, `/api/health`, `/api/scheduler/status`.

> НЕ викликати POST-маршрути (toggle/scan/delete-timer/force-open/settings/
> trade-close). Сайт лише читає.

## Що показувати (рекомендовано — всі блоки)

1. **❤️ Черга на вхід + ⏱ таймери** — символ, напрямок, виснаженість, скільки
   очікує, ММ-напрямок.
2. **Угоди** — відкриті позиції з таймером; історія/архів закритих з Reason
   (пік/віддано, exhaust вхід→вихід).
3. **💰 Funding-монети** — ставка, countdown до виплати, прогрес entry→−4%,
   Vol 24h, стрілка тренду (←зменшується / →збільшується).
4. **Статус BTC/двигуна** — банер ₿ BTCUSDT (ММ-напрямок), START/STOP,
   загальний стан бота, health.

## Вимоги

- Автооновлення (polling, напр. кожні 3–5 с) без «дьоргання» екрана.
- Українська мова інтерфейсу.
- Адаптивна верстка (зручно з телефона).
- Чітка індикація «дані недоступні», якщо бот не відповідає (не падати).
- Тільки read-only: жодних кнопок, що шлють запити-дії боту.

## Чого НЕ робити

- Не редагувати код бота й шаблони бота.
- Не дублювати бізнес-логіку (рахунок палива/вердиктів) — лише показ готових
  даних з API.
- Не звертатись напряму до Bybit/бірж — усе через API бота.
