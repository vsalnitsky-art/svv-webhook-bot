### 1. ХРАНЕНИЕ ДАННЫХ

| Аспект | app.py (СТАРЫЙ) | main_app.py (НОВЫЙ) |

## 🎯 ПРАКТИЧЕСКИЙ ПРИМЕР

### Задача: Показать топ-5 монет с наибольшим вливанием за последние 6 часов

#### app.py (СТАРЫЙ КОД):
```python
def get_top_coins_old():
    # 1. Перебрать ВСЕ сигналы
    coins = {}
    cutoff = datetime.now() - timedelta(hours=6)
    
    for alert in pump_alerts:  # ❌ O(n)
        if alert['timestamp'] > cutoff:
            symbol = alert['symbol']
            if symbol not in coins:
                coins[symbol] = 0
            coins[symbol] += alert['vol_inflow']
    
    # 2. Отсортировать
    sorted_coins = sorted(
        coins.items(), 
        key=lambda x: x[1], 
        reverse=True
    )[:5]  # ❌ O(n log n)
    
    return sorted_coins
    # Сложность: O(n log n)
    # Проблема: Перебирает ВСЕ данные в памяти
```

#### main_app.py (НОВЫЙ КОД):
```python
def get_top_coins_new():
    # Одна SQL команда с индексами
    coins = stats_service.get_top_coins(
        period_hours=6,
        limit=5,
        sort_by='inflow'
    )
    return coins
    # Сложность: O(log n)
    # SQL: SELECT symbol, SUM(volume_inflow) 
    #      FROM whale_signals 
    #      WHERE timestamp > ?
    #      GROUP BY symbol
    #      ORDER BY SUM(volume_inflow) DESC
    #      LIMIT 5
    # ✅ Использует индексы, работает в 10-100x быстрее
```
